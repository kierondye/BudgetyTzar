# Agent Guidance

Use the repository's human-facing documentation as the source of truth:

- [SPECIFICATION.md](SPECIFICATION.md) defines product rules and externally
  observable behaviour.
- [CONTRIBUTING.md](CONTRIBUTING.md) defines the development workflow and the
  concise domain, persistence, application, and testing patterns.
- [docs/architecture.md](docs/architecture.md) explains the code structure,
  responsibility boundaries, request flow, rationale, and worked examples.

Read the relevant sections before making a change. Do not duplicate their guidance
here. If implementation and specification disagree, treat the specification as
authoritative unless the task explicitly changes it.
